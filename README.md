# web_bs3_origin → web_bs5 本次项目内的实际升级点（两个仓库横向对比）
## 一、框架底层核心升级
1. **Bootstrap 3 升级为 Bootstrap 5**
   - 保留了原有全部业务页面：`INDEX.HTM`、`DCU_VIEW.HTM`、`general_view.htm`、`alarm_data.htm` 等页面结构、业务逻辑、多语言配置完全复用，只做UI样式类名改造。
   - 仓库里保留了 `bootstrap.min.css.bs3.bak` 作为旧版本备份，用来做回归比对，防止样式错乱。
   - 彻底移除了 Bootstrap 3 老旧浏览器兼容文件：删掉了 `ie10-viewport-bug-workaround.css`，不再兼容IE低版本。

2. **JS 依赖架构变更**
   - BS3 强依赖 jQuery，页面大量基于 jQuery 选择器、事件、DOM操作；
   - BS5 不再依赖 jQuery，原有 jQuery 相关交互逻辑全部改为**原生 JavaScript**重写；
   - 下拉、弹窗、提示框等组件底层依赖 Popper，替代了原来 jQuery 实现的定位逻辑，修复了旧项目下拉溢出、弹窗位置错乱问题。

## 二、样式层改造升级
1. **类名大规模迁移适配**
   - 栅格、表单、导航、按钮、标签页（tabcontrol）等组件全部替换 BS5 标准类名；
   - 保留项目自研业务样式：`component.css`、`dashboard.css`、`justified-nav.css`、`tabcontrol.css` 全部迁移复用，只修改和BS3冲突的样式选择器。
2. **布局能力增强**
   - 借助BS5弹性工具类重构了仪表盘、设备数据面板布局，解决BS3下复杂表格、多卡片布局错位问题；
   - 利用BS5原生间距、阴影、圆角工具类精简了大量自定义CSS代码。

## 三、业务资源完全保留（无业务功能删减）
1. **国际化能力原样迁移**
   - `LANG` 目录三套语言包：中文、英文、法语文案 JSON 文件直接复用，多语言切换逻辑只适配BS5组件事件，业务文案没有改动。
2. **设备安全与固件相关目录完全继承**
   - `CFG` 证书、密钥、安全配置、配置备份文件夹完整保留；
   - `APL` 固件镜像、固件数字签名校验文件、`BIN` 二进制工具文件全部沿用；
   - HTTPS公私钥、系统运行日志 `sysLog.txt`、设备模拟数据 `data.json` 均未修改，保证设备固件升级、配置备份、加密通信功能和旧版完全一致。
3. **页面业务结构不变**
   登录首页、DCU设备监控页、告警记录页、系统通用参数页四大核心业务页面的功能、表格、数据展示逻辑、按钮操作全部保留，仅做UI层升级。

## 四、兼容性与可维护性升级
1. 放弃老旧IE浏览器支持，页面渲染速度提升，减少大量兼容冗余样式；
2. BS5支持CSS变量，后续主题色、全局样式可以统一配置，不用像BS3那样大范围修改样式文件；
3. 标签页、告警弹窗、数据表格这些高频业务组件，用BS5原生组件重构，替换了BS3时代大量自定义封装的JS控件，降低后续BUG概率；
4. 静态图片资源 `IMG` 目录全部复用，品牌、设备图标无需重新设计。

## 五、本次重构没有改动的部分
1. 后端交互方式、数据格式（`data.json`）不变；
2. 多语言数据结构不变；
3. 设备证书、固件、安全配置等运维相关文件完全不变；
4. 整体业务流程：登录→设备查看→告警查询→配置备份→固件升级 完全一致。

## 精简总结本次项目实际升级
1. UI框架：BS3 → BS5，删除IE兼容，保留所有自研业务CSS；
2. 脚本层：jQuery 全部迁移为原生JS，依赖简化；
3. 业务层：页面、多语言、设备配置、固件、安全加密**全部原样继承**，只做前端交互与样式适配；
4. 可维护性提升：标准化BS5组件，精简自定义JS、CSS，方便后续用TS进一步工程化重构。
